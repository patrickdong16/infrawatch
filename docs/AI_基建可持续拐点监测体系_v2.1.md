# AI 基建可持续拐点监测体系 v2.1

**修订日期**: 2026年1月  
**版本说明**: v2.1 新增E板块（供应链价格信号），覆盖HBM/DRAM等上游成本

---

## 一、监测体系设计原则

### 核心纪律

| 原则 | 执行要求 |
|------|----------|
| 只抓一手数据 | 仅采集官方披露、定价页、财报原文、行业数据库 |
| 不估算 | 用官方区间做敏感性分析，不做点估计 |
| 不预测 | 只记录已发生的变化，不推演未来 |
| Agent不下判断 | 系统只输出信号，阶段判定由人工确认 |

### 因果链条（更新版）

```
上游供应链成本 (E板块 - HBM/DRAM/封装)
        ↓
算力供需平衡 (C板块 - 云服务价格)
        ↓
推理现金流 (B板块 - 模型API价格)
        ↓
基建自养能力 (M01代理指标)
        ↓
应用层真实商业价值 (A板块)
        ↓
巨头财务健康度 (D板块)
```

### E板块：供应链价格信号（新增）

**核心逻辑**：HBM和先进封装是AI算力成本的关键上游，价格波动直接影响GPU成本和基建投资回报。监测供应链价格可提前6-12个月预判算力成本变化。

| metric_id | 指标名称 | 监测对象 | 来源 | 频率 |
|-----------|----------|----------|------|------|
| E1 | HBM3e 合约价格 | SK海力士、三星、美光 HBM3e 报价 | TrendForce, DRAMeXchange | 月度 |
| E2 | 服务器DRAM价格指数 | DDR5 服务器内存现货价 | DRAMeXchange, 集邦科技 | 周度 |
| E3 | NVIDIA GPU ASP估算 | H100/H200/B200 平均售价 | 分析师报告、渠道调研 | 季度 |
| E4 | 台积电CoWoS产能利用率 | 先进封装产能紧张度 | 供应链调研、分析师报告 | 季度 |
| E5 | HBM出货量增速 | SK海力士/三星HBM出货同比 | 财报、行业报告 | 季度 |

**数据采集模板 (E_supply_chain_pricing.csv)**

```csv
date,category,item,price,unit,mom_change,yoy_change,source,notes
2026-01-15,HBM,HBM3e-24GB,15.50,USD/GB,+3.2%,+45%,TrendForce,供应紧张
2026-01-15,DRAM,DDR5-4800-64GB,185.00,USD/piece,-2.1%,-15%,DRAMeXchange,服务器需求稳定
2026-01-15,GPU,H100-80GB-SXM,28500,USD/unit,0%,-12%,渠道调研,渠道价趋稳
2026-01-15,GPU,H200-141GB,35000,USD/unit,-5%,N/A,渠道调研,新品上量
2026-01-15,Packaging,CoWoS利用率,98,%,-,+8pp,供应链调研,持续满载
```

**派生指标**

| 派生指标 | 计算方式 | 意义 |
|----------|----------|------|
| E_hbm_premium | HBM3e单位价格 / DDR5单位价格 | HBM溢价倍数，反映AI专用内存稀缺度 |
| E_memory_cost_index | HBM + DRAM加权价格指数 | GPU内存成本综合指标 |
| E_supply_tightness | CoWoS利用率 × HBM涨幅 | 供应链紧张度综合评分 |
| E_gpu_margin_proxy | GPU ASP变化 - 内存成本变化 | GPU厂商毛利空间变化代理 |

**解读规则**

| 信号组合 | 含义 | 影响传导 |
|----------|------|----------|
| E1↑ 且 E4>95% | HBM供应极度紧张 | 6-12月后GPU成本上升，C板块价格承压 |
| E1↓ 且 E4<85% | HBM供应转松 | 6-12月后GPU成本下降，利好基建投资 |
| E3↓ 且 E1稳定 | NVIDIA让利抢市场 | GPU厂商毛利承压，但下游受益 |
| E5>50% YoY | HBM需求爆发 | 验证AI训练/推理需求真实性 |

**与其他板块联动**

```
E板块供应链价格
    │
    ├──→ C板块GPU租赁价格（6-12月滞后）
    │        供应链成本上升 → 租赁价格上涨压力
    │
    ├──→ D板块Capex效率
    │        HBM涨价 → 同等Capex购买力下降
    │
    └──→ M01覆盖率
             成本上升 → 折旧增加 → M01承压
```

---

### M01代理指标：推理覆盖率估算

**核心公式**: M01 = AI推理收入 / AI相关资产年化折旧

| M01 区间 | 状态判定 |
|----------|----------|
| < 0.3 | 严重补贴期，高风险 |
| 0.3 - 0.7 | 过渡期，需观察趋势 |
| 0.7 - 1.0 | 接近自养，健康 |
| > 1.0 | 完全自养，可持续 |

## 三、阶段判定规则

### 触发条件（v2.1修订版）

| 阶段 | 触发条件 |
|------|----------|
| **S0** | M01_high < 0.3 **且** B_qoq_deflation > 15% **且** C_spot_discount > 40% |
| **S1** | M01 在 0.3-0.7 区间 **或** A1/A2/A3 连续两季正增长 |
| **S2** | M01_low > 0.7 **且** C_rental_index 环比变化 < ±5% **且** E_supply_tightness 稳定 |
| **S3** | M01_low > 1.0 **且** B_price_index 下降 **但** D3 云利润率稳定 |

### 供应链预警信号（新增）

| 预警类型 | 触发条件 | 影响评估 |
|----------|----------|----------|
| 成本上行预警 | E1 连续3月上涨 >5% 且 E4 >95% | 6-12月后基建成本上升，可能压制M01 |
| 供应缓解信号 | E1 环比下跌 >10% 或 E4 <85% | 未来成本压力减轻，利好Capex效率 |
| 需求验证信号 | E5 YoY >50% 且 E1 上涨 | AI需求真实强劲，供应链被拉动 |

## 四、信号类型定义

| signal_type | 定义 | 触发条件示例 |
|-------------|------|-------------|
| `price_move` | 价格显著变动 | B或C板块指标环比变化 > 10% |
| `adoption_inflection` | 采用率拐点 | A板块指标环比加速/减速 > 20% |
| `coverage_threshold` | 覆盖率跨越阈值 | M01 跨越 0.3/0.7/1.0 边界 |
| `supply_demand_shift` | 供需格局变化 | C_spot_discount 变化 > 10pp |
| `disclosure_change` | 披露口径变化 | 厂商停止/开始披露某指标 |
| `supply_chain_alert` | 供应链异动（新增） | E1/E4 显著变化，触发成本预警 |
